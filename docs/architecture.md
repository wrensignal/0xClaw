# WrenOS Architecture

WrenOS is a composable operator control plane for crypto-agent deployments. The architecture is designed so operators can start in paper mode quickly, then progressively harden for production.

## Repository boundary (what this repo is)

This repository is the open-source **WrenOS control plane**.
It includes:
- WrenOS CLI
- profiles and packs
- loop/adaptation primitives
- adapters
- inspectable file-based operator config

It can run fully self-hosted, and can also integrate with hosted-default runtime/inference services (for example Speakeasy).

## Layer model

1. **Profiles (`packages/profiles`)**
   - Declarative operator intent (research-only, paper trading, live-disabled, meme-discovery pack templates).
   - Encodes defaults for inference routing, loop behavior, data sources, and execution constraints.

2. **Adapters (`packages/adapters`)**
   - Runtime connectors for:
     - inference (Speakeasy + OpenAI-compatible request shape)
     - execution venues (Jupiter + pump.fun)
     - operator interface (Telegram command adapter)
   - Keeps integration logic out of strategy code.

3. **Loops (`packages/loops`)**
   - Heartbeat and regression primitives.
   - Intended to enforce adaptive cadence + safety gates before external side effects.

4. **CLI control plane (`packages/cli`)**
   - Operator bootstrap and local runtime configuration.
   - `init`, `init-pack`, `doctor`, `status`, `config set`, `wallet setup`, `test inference`, `test execution`, `bootstrap-wrenos`.

5. **Vendored MCP and skills (`vendor/*`, `vendor/skills/*`)**
   - Curated local dependencies for deterministic setup.
   - `.mcp.json` template generated by CLI wires `agenti-lite`, `pump-fun-sdk-lite`, and `helius`.

## Runtime data flow

### Inference path

`agent loop -> adapters/inference -> Speakeasy endpoint -> model route`

- Default endpoint: `https://api.speakeasyrelay.com`
- Route families:
  - research
  - deep_think
  - codegen
  - uncensored

### Execution path

`signal decision -> adapters/execution -> venue quote/build -> paper/live policy gate`

- Supported venues in default adapter:
  - Jupiter (referral-capable)
  - pump.fun (direct bonding-curve path)

### Research-to-trading handoff

For dual-agent packs, research briefs and meme watchlists are written to inbox JSON contracts consumed by trading workflows.

## Deployment model

- Local bootstrap via `scripts/install.sh` + CLI init
- Railway template and deploy notes included
- Open source core with hosted-default routing path

## Design principles

- **Safe-by-default:** live execution off unless explicitly enabled.
- **Evidence-first operations:** health/test commands expose concrete readiness checks.
- **Composable overrides:** operators can replace endpoints, models, venues, and pack configs.
- **Operator trust:** config is file-based, inspectable, and reproducible.

          - region "Template Documentation" [ref=e118]:
            - heading "Deploy and Host WrenOS on Railway" [level=1] [ref=e119]
            - paragraph [ref=e120]: WrenOS is an open-source control plane for operator-managed crypto agents. It provides a paper-first workflow for discovery, scoring, validation, and execution with explicit safety gates and inspectable artifacts. This template gives you a fast hosted path to run WrenOS with your own keys, profile, and operator controls.
            - heading "About Hosting WrenOS" [level=2] [ref=e121]
            - paragraph [ref=e122]: Hosting WrenOS on Railway gives you a managed runtime for continuous agent operations without self-managing infrastructure. You configure environment variables, choose a profile, and run health/bootstrap checks to validate readiness. WrenOS is designed to start in paper-first mode and fail closed when required signals are missing, so operators can verify behavior before enabling execution-capable flows. This template supports Telegram-based operator UX, optional data-provider enrichment (Helius, Birdeye, LunarCrush, Jupiter), and clean post-deploy validation using built-in preflight and health scripts. The result is a reproducible hosted setup that is safer and easier to operate than ad hoc local-only workflows.
            - heading "Common Use Cases" [level=2] [ref=e123]
            - list [ref=e124]:
              - listitem [ref=e125]: Run paper-first crypto research and decision workflows with operator visibility and safety defaults.
              - listitem [ref=e126]: Host an always-on control plane for Telegram alerts, commands, and monitoring loops.
              - listitem [ref=e127]: Stand up a reproducible environment for testing profile configurations and release-readiness checks.
            - heading "Dependencies for WrenOS Hosting" [level=2] [ref=e128]
            - list [ref=e129]:
              - listitem [ref=e130]: A Railway account and a deployed template/service from this repository.
